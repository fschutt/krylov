# Krylov Subspace Iterative Solvers in Rust (`krylov_rs`)

`krylov_rs` is a Rust library providing implementations of common Krylov subspace iterative methods for solving large, sparse linear systems of equations, particularly `Ax = b`. These methods are essential when the matrix `A` is too large for direct factorization or when it's only available as a matrix-free linear operator.

## Overview

Krylov subspace methods iteratively refine a solution by constructing an orthonormal basis for the Krylov subspace generated by `A` and the initial residual `r_0 = b - Ax_0`. The approximate solution is then found within this subspace by imposing conditions like orthogonality of the residual to the subspace (as in GMRES) or bi-orthogonality (related to BiCG methods).

## Features

- **Generic Implementation**: Solvers are generic over the scalar type `S` (e.g., `f32`, `f64`) and abstract linear operators.
- **Matrix-Free Operations**: Define your linear operator by implementing the `LinearOperator<S>` trait, allowing `A` to be a dense matrix, a sparse matrix, or a function that computes `Ax`.
- **Preconditioning**: Support for (right) preconditioning to accelerate convergence via the `Preconditioner<S>` trait.
- **Common Solvers**:
    - **GMRES (Generalized Minimal Residual method)**: Effective for general non-symmetric systems. Includes restarts to manage computational resources.
    - **BiCGSTAB (Biconjugate Gradient Stabilized method)**: A popular choice for non-symmetric systems, often converging faster than GMRES and without needing `A^T`.

## Getting Started

Add `krylov_rs` to your `Cargo.toml`:

```toml
[dependencies]
krylov_rs = "0.1.0" # Replace with the desired version
ndarray = "0.15"    # Required for vector types
num-traits = "0.2"  # Required for scalar traits
# Ensure your scalar type (e.g., f64) is compatible with LinalgScalar and Float.
```

*(Note: The version "0.1.0" is a placeholder; use the actual version from crates.io if published, or a git dependency if using a repository.)*

## Usage Example

Here's a basic example using GMRES. BiCGSTAB usage is very similar.

```rust
use krylov_rs::gmres::{gmres, GmresParams};
use krylov_rs::operator::LinearOperator;
use ndarray::{array, Array1, Array2, LinalgScalar};
use num_traits::{Zero, NumAssign, Float, FromPrimitive, Signed, PartialOrd};
use std::iter::Sum;
use std::ops::{AddAssign, Neg};
use ndarray::ScalarOperand;
use std::fmt::Debug;

// Define a simple matrix operator for demonstration
struct MyMatrix<S: LinalgScalar + NumAssign> {
    matrix: Array2<S>,
}

impl<S: LinalgScalar + NumAssign> LinearOperator<S> for MyMatrix<S> {
    fn rows(&self) -> usize { self.matrix.nrows() }
    fn cols(&self) -> usize { self.matrix.ncols() }
    fn apply(&self, x: &Array1<S>) -> Array1<S> { self.matrix.dot(x) }
    fn apply_adjoint(&self, x: &Array1<S>) -> Array1<S> { self.matrix.t().dot(x) }
}

fn main() -> Result<(), String> {
    type Scalar = f64;

    let matrix_a = MyMatrix {
        matrix: array![[4.0, 1.0, 0.0], [1.0, 3.0, 1.0], [0.0, 1.0, 2.0]],
    };
    let b: Array1<Scalar> = array![1.0, 2.0, 3.0];
    let x0: Array1<Scalar> = Array1::zeros(matrix_a.cols());

    let params = GmresParams {
        restart_length: 3,
        max_iterations: 10,
        tolerance: 1e-9,
    };

    match gmres(&matrix_a, &b, &x0, &params, None::<&MyMatrix<Scalar>>) {
        Ok(solution) => {
            if solution.converged {
                println!("GMRES converged in {} iterations.", solution.iterations);
                println!("Solution x: {:?}", solution.x);
                println!("Final residual norm: {:?}", solution.residual_norm);
            } else {
                println!("GMRES did not converge. Reason: {}", solution.reason);
            }
        }
        Err(e) => eprintln!("Error during GMRES: {}", e),
    }
    Ok(())
}
```
